1. HTML은 “웹페이지의 뼈대”

교수님은 HTML을 그냥 태그 암기 과목처럼 보기보다, 콘텐츠를 구조화하는 언어로 이해하라고 했어.
중요한 건 “웹페이지를 왜 만드는가”까지 말할 수 있어야 한다는 점이야. 웹페이지에는 텍스트, 이미지, 오디오, 비디오 같은 콘텐츠가 들어가고, 그것을 목적에 맞게 구성하는 게 HTML이라고 설명했어.

2. 웹페이지를 왜 만드는지 설명할 수 있어야 함

교수님이 계속 강조한 건 **“왜 웹페이지를 만드는지 말할 수 있어야 한다”**는 거였어.
단순히 코드를 짜는 게 아니라, 기업용인지, 플랫폼용인지, 홍보용인지, 예약/업무처리용인지 구분해서 생각해야 하고, 나중에 면접에서도 이런 질문이 기본적으로 나올 수 있다고 했어. 그래서 조카에게 설명하듯 말하는 연습을 하라고 했어. 

붙여넣은 텍스트 (1)

3. 메타데이터와 SEO도 실무에서 중요

메타데이터는 HTML의 head 안에 들어가는 문서 설명 정보라고 설명했어.
실무에서는 검색엔진 노출, AI 노출, 검색 최적화 때문에 메타데이터를 신경 써야 하고, SEO도 기본 상식으로 알아야 한다고 했어. 특히 제목(title) 같은 정보가 검색 노출에 영향을 준다는 식으로 설명했어. 

붙여넣은 텍스트 (1)

교수님은 HTML/CSS를 공부할 때 표준 문서와 공식 문서를 보는 습관이 중요하다고 했어.
W3C, MDN 같은 사이트를 훑어보는 것만으로도 큰 차이가 난다고 했고, 풀스택을 지향하면 이런 자료는 기본적으로 봐야 한다고 말했어.

Emmet은 HTML/CSS 작성 속도를 높여주는 자동완성 도구라서, 실습할 때 계속 쓰게 될 거라고 했어.

6. MPA와 SPA는 직접 만들어보며 구분

교수님 설명의 핵심은 이거였어.

MPA: 페이지가 여러 개이고, 클릭할 때마다 전체 페이지가 바뀌는 방식

SPA: 한 페이지 안에서 일부 영역만 바뀌는 방식

예시로 페이스북 같은 건 SPA 쪽으로 볼 수 있다고 했고, 학교 홈페이지처럼 여러 페이지가 이동하는 구조는 MPA로 이해하면 된다고 했어. 그리고 이건 말로만 외우지 말고 직접 파일 만들어서 차이를 확인하라고 했어. 

붙여넣은 텍스트 (1)

8. Node.js는 플랫폼보다는 JavaScript 실행 환경

10. npm과 npx 차이

11. React를 배우는 이유

교수님은 “HTML/CSS/JS만으로도 페이지는 만들 수 있는데 왜 React를 배우냐”는 질문을 직접 던졌어.
답은 속도와 효율 때문이라고 설명했어. React는 가상 DOM을 이용해서 바뀐 부분만 화면에 반영하므로, 많은 사용자가 동시에 접속하는 서비스에서 유리하다고 했어. 다만 모든 사이트가 무조건 React가 필요한 건 아니고, 상황에 따라 선택할 줄 아는 것이 중요하다고 했어.
